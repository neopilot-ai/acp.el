Installation

Get acp.el running in your Emacs in minutes.

Requirements

PackageVersionDescription
Emacs29.1+Core requirement
shell-maker0.89.2+Shell interface layer
acp0.11.1+ACP protocol client

Via MELPA (Recommended)

The easiest way to install acp.el is through MELPA.

use-package

(use-package acp
  :ensure t
  :config
  ;; Optional: set your preferred agent
  (setq acp-preferred-agent-config
        (acp-anthropic-make-claude-code-config)))

straight.el

(straight-use-package 'acp)

Manual (package.el)

M-x package-install RET acp RET

Manual Installation

Clone the repository and add the directories to your load-path:

git clone https://github.com/neopilot-ai/acp.el.git ~/acp.el

Add to your Emacs config (~/.emacs.d/init.el or ~/.emacs):

(add-to-list 'load-path "~/acp.el/agents")
(add-to-list 'load-path "~/acp.el/ui")
(add-to-list 'load-path "~/acp.el/features")
(add-to-list 'load-path "~/acp.el")
(require 'acp)

Authentication

Each agent requires authentication. Configure at least one agent to get started.

Anthropic (Claude Code)

;; Option 1: API key
(setq acp-anthropic-authentication
      (acp-anthropic-make-authentication :api-key "sk-ant-..."))

;; Option 2: Browser login
(setq acp-anthropic-authentication
      (acp-anthropic-make-authentication :login t))

Google (Gemini CLI)

;; Uses Application Default Credentials
;; Run: gcloud auth application-default login
(setq acp-google-authentication
      (acp-google-make-authentication :login t))

OpenAI (Codex)

;; API key
(setq acp-openai-authentication
      (acp-openai-make-authentication :api-key "sk-..."))

See the Configuration page for detailed authentication options.

Verify Installation

After configuring, start Emacs and run:

M-x acp

You should see the agent selection prompt. Select an agent and start chatting.

Troubleshooting

Agent binary not found

Ensure the agent CLI is installed and on your PATH:

;; Check if the binary exists
(which claude)    ;; For Claude Code
(which gemini)    ;; For Gemini CLI
(which codex)     ;; For Codex

shell-maker version mismatch

acp.el requires shell-maker 0.89.2+. Update via M-x package-upgrade.
