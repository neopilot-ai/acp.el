Getting Started

1. Clone the Repository

git clone https://github.com/neopilot-ai/acp.el.git
cd acp.el

2. Add to load-path

(add-to-list 'load-path "/path/to/acp.el")
(add-to-list 'load-path "/path/to/acp.el/agents")
(add-to-list 'load-path "/path/to/acp.el/ui")
(add-to-list 'load-path "/path/to/acp.el/features")

3. Run Tests

# Load check
make test

# Run ERT tests
make test-standalone

# Byte-compile lint
make lint

# Clean compiled files
make clean

Feature Module Pattern

(eval-when-compile (require 'cl-lib))
(require 'map)

(declare-function acp--state "acp")
(declare-function acp--dot-subdir "acp")

(defgroup acp-my-feature nil
  "My feature for acp."
  :group 'acp)

(defcustom acp-my-feature-option t
  "Enable my feature."
  :type 'boolean
  :group 'acp-my-feature)

(defun acp-my-feature--helper ()
  "Internal helper function.")

;;;###autoload
(defun acp-my-feature-command ()
  "Interactive command."
  (interactive)
  ...)

(provide 'acp-my-feature)
;;; acp-my-feature.el ends here

Test Pattern

(require 'ert)
(require 'acp-my-feature)

(ert-deftest acp-my-feature--helper/test-case ()
  "Description of the test scenario."
  (let ((result (acp-my-feature--helper)))
    (should (equal result "expected"))))

(ert-deftest acp-my-feature-command/error-case ()
  "Test that an error is signaled."
  (should-error (acp-my-feature-command)
                :type 'error))

Adding a New Agent

  1. Create agents/acp-myagent.el
  2. Implement acp-myagent-make-config function
  3. Implement acp-myagent-make-client function
  4. Add tests in tests/acp-myagent-tests.el
  5. Register in acp--make-default-agent-configs in acp.el
  6. Update README.md and docs/agents.html

Adding a New Feature

  1. Create features/acp-myfeature.el
  2. Follow the feature module pattern above
  3. Add tests in tests/acp-myfeature-tests.el
  4. Add require in acp.el: 
    (require 'acp-myfeature "features/acp-myfeature")
  5. Update documentation

Pull Request Guidelines

  • Write your own PR descriptions — don't have AI generate them
  • Review all code in your PR and vouch for its quality
  • Run make test and make lint before submitting
  • Add tests for new functionality
  • Keep commits focused and messages concise
  • Follow existing code style and conventions

Reporting Issues

When reporting bugs, please include:

  • Emacs version (M-x emacs-version)
  • acp.el version
  • Agent being used
  • Steps to reproduce
  • Expected vs actual behavior
  • *Backtrace* if applicable (M-x toggle-debug-on-error)
